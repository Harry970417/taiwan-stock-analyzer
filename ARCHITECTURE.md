## 核心設計原則

| 原則 | 說明 |
|------|------|
| **T+1 執行** | 今日訊號 → 隔日開盤執行，避免未來函數偏誤 |
| **信心度雙重門檻** | 低信心資料不給強烈買入建議，防止過度自信 |
| **資料來源透明** | 每筆資料標記 Tier 等級、延遲時間、更新時間 |
| **N/A 優先於假資料** | 無法取得資料一律顯示 N/A，不用假數字填補 |
| **整張交易** | 最小單位 1000 股（1 張），符合台股規則 |
